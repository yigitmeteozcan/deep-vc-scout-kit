# Setting Up Your ChatGPT Project

This guide walks you through creating the ChatGPT Project and uploading your context pack.

---

## Prerequisites

- ChatGPT Plus, Pro, or Team subscription (Deep Research requires Plus or above)
- `private/context_pack.md` generated by `python scripts/build_context_pack.py`

---

## Step 1: Create a ChatGPT Project

1. Go to [chatgpt.com](https://chatgpt.com)
2. In the left sidebar, click **"New project"** (or look for the Projects section)
3. Name the project something like: `Deep VC Scout — [Your Fund Name]`
4. Click **Create**

Projects persist across chats. Every new chat inside the project shares the same uploaded files and instructions.

---

## Step 2: Upload the context pack

1. Inside your new project, look for **"Add files"** or the file upload option in the project settings
2. Upload `private/context_pack.md` from your local machine

The context pack will now be available to every chat inside this project. GPT will read it at the start of each scouting session.

**Important:** If you regenerate the context pack (e.g., after updating your pipeline or fund thesis), re-upload the new version to the project. Delete the old version if the project shows both.

---

## Step 3: Set project instructions (optional but recommended)

Some ChatGPT plans allow you to add persistent instructions to a project.

If available, paste the contents of `gpt-project/project-instructions.md` into the project's custom instructions field.

This ensures GPT always follows the source-first scouting methodology without needing it in every prompt.

---

## Step 4: Start your first scouting session

1. Open a **new chat** inside the project (not a new project — a new chat within the same project)
2. Paste the contents of `gpt-project/deep-research-master-prompt.md`
3. Customize the prompt with:
   - Your target geography for this session
   - Your target sector focus for this session
   - Any specific constraints or source paths to prioritize
4. Send the message and let Deep Research run

Deep Research typically takes 5-15 minutes. When it finishes, it will return a structured output with sources checked, rejected candidates, and final candidates in CSV format.

---

## Step 5: Save GPT's CSV output

When GPT returns the final candidates CSV:

1. Copy the CSV block from the chat
2. Save it as `private/new_candidates.csv` on your local machine
3. Make sure the header row matches the output schema:
   ```
   company_name,website,source_url,short_description,founder_signal,geo_angle,recent_signal,traction_proof,why_non_obvious,why_fund_should_care,sector,source_type
   ```

If GPT returned the data without a header, add the header manually.

---

## Step 6: Run the local pipeline

```bash
python scripts/dedupe_candidates.py
python scripts/score_candidates.py
python scripts/review_candidates.py
python scripts/generate_notes.py
```

Your analyst notes will be in `private/analyst_notes.md`.

---

## Tips for better Deep Research results

**Be specific in your session prompt:**
Instead of "scout for industrial SaaS startups in the Baltics," say:
"Scout for pre-seed industrial maintenance software companies in Estonia, Latvia, and Lithuania founded in 2022-2024. Prioritize founders with operator backgrounds at manufacturing or logistics companies. Start with university spinout pages, EIT program cohorts, and Estonian/Latvian grant recipient databases. Avoid mainstream startup media."

**Run separate sessions for each search mission:**
One session for grant/university sources. One session for LinkedIn/hiring signals. One session for conference exhibitors. This keeps each session focused and avoids dilution.

**Ask GPT to list sources checked:**
If GPT does not include the sources checked section, ask: "List all source paths you searched and what you found at each."

**Ask for rejected candidates:**
If GPT does not include the rejected candidates section, ask: "List every company you researched that you did not include, with the specific rejection reason."

---

## Keeping the project up to date

- After each scouting session, update `private/pipeline.csv` with newly known companies
- Regenerate the context pack: `python scripts/build_context_pack.py`
- Re-upload the new `private/context_pack.md` to the ChatGPT Project
- Your fund's memory grows with each session
